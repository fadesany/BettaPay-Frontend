# Issue #110: Add transaction signing UI feedback

## Branch Name
`feature/110-transaction-signing-feedback`

## Commit Message
`feat(stellar): add transaction signing progress UI feedback in-progress`

## Issue Description
When the user initiates a Stellar transaction (e.g., settlement), there's no visual feedback during the signing process. The user sees a brief loading state but doesn't know what's happening (waiting for Freighter popup, waiting for Horizon confirmation, etc.).

## Requirements
1. Create a transaction status tracker component showing the current step:
   - "Waiting for Freighter" — Freighter popup is open
   - "Signing transaction" — User approved in Freighter, waiting for signature
   - "Submitting to network" — Transaction sent to Horizon
   - "Confirming settlement" — Waiting for blockchain confirmation

2. Show an animated progress indicator through the steps
3. Handle each step's failure with specific error messages

## Implementation Plan

### 1. Component Creation
- **File**: `components/stellar/TransactionProgress.tsx`
- **Props**:
  - `transactionTask`: Promise or async function representing the transaction
  - `onComplete`: Callback when transaction completes successfully
  - `onError`: Callback when transaction fails
- **State Management**:
  - `currentStep`: Tracks which of the 4 steps is active
  - `status`: 'pending' | 'active' | 'complete' | 'error'
  - `errorMessage`: Specific error message for failed steps

### 2. Step Definition
Create enum for transaction steps:
```typescript
enum TransactionStep {
  WAITING_FREIGHTER = "Waiting for Freighter",
  SIGNING = "Signing transaction",
  SUBMITTING = "Submitting to network",
  CONFIRMING = "Confirming settlement"
}
```

### 3. UI Design
- Progress indicator with 4 connected steps
- Animated transitions between steps (using CSS or framer-motion)
- Status indicators (pending → active → complete → error)
- Error display with step-specific messages
- Brand colors matching BettaPay design system

### 4. Integration Points
- Settlement initiation flow (primary use case)
- Any other Stellar transaction flows
- Transaction status tracking with real-time updates

### 5. Error Handling
- Freighter not installed/connected
- User rejects transaction
- Network submission failures
- Confirmation timeouts
- Step-specific error messages with recovery options

### 6. Animation Plan
- Smooth progress bar animation
- Step transition animations
- Status icon animations (spinner for active, checkmark for complete, X for error)

### 7. Testing Strategy
- Unit tests for component rendering and state transitions
- Integration tests with actual transaction flows
- Error scenario testing
- Responsive design testing

### 8. Brand Compliance
- Use BettaPay color palette
- Follow existing design patterns
- Maintain accessibility standards
- Responsive design for all screen sizes
